Описание решения NUKE:

* мы не используем double и float ! (используем арифметику с фиксированной точкой)

* генерируем таблицу для следующей формулы:
  q = sqrt((dx)^2 + (dy)^2) , sqrt(r^2-(q/2)^2)*(dy)/q и sqrt(r^2-(q/2)^2)*(dx)/q по всем возможным значениям dx = 0..99, dy=dx..99, R=1..71
  получается файл размером примерно 1 Мбайт
  
* по этой таблице/файлу мы для всех заданных пар точек вычисляем сдвиг центра (соответственно сопроцессор нам не нужен)

* затем для каждого возможного центра считаем количество целей, которое попадает в круг поражения с заданным радиусом (это не требует сопроцессора, так как используются только операции + и *)

NOTE: алгоритмическая сложность решения O(N^3). без float, double и сопроцессора

NOTE: для того, чтобы решение работало на 16битовых, 32битовых и прочих моделях ноута считываем из таблицы символы fixed point значений последовательно, выводим решение с информационной записью, что при повышенной точности решение может быть лучше (NB!)

* epsilon (fixed_epsilon) найден эмпирическим путем (можно так же использовать аналитические расчеты или комбинированный метод)
